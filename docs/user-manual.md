## 1. Getting Started — Logging In

### What You Need

- A computer, tablet, or smartphone with a modern web browser (Chrome, Firefox, Edge, or Safari)
- An internet connection (or see Section 7 for offline mode)
- Your username and password (provided by your coordinator or the system administrator)

### Steps to Log In

1. Open your web browser and go to: **https://merl.climate.gov.vu**

2. You will see the MERL Dashboard login page. Click the **"Sign In"** button.

3. You will be taken to the secure login screen. Enter your **username** and **password**.

   > If this is your first login, you may be asked to set a new password. Choose a strong password that is at least 8 characters long and includes numbers and letters.

4. Click **"Log In"**.

5. You will be returned to the MERL Dashboard and taken to your home dashboard.

### Logging Out

Always log out when you finish using the system, especially on shared computers.

1. Click your name or profile icon in the top right corner of the screen.
2. Click **"Log Out"**.

### Forgotten Password

If you forget your password:

1. On the login screen, click **"Forgot Password?"**
2. Enter your email address.
3. Check your email for a password reset link.
4. Click the link and follow the instructions to set a new password.

If you do not receive the email within 10 minutes, contact your coordinator or the system administrator.

## 3. Submitting Indicator Values

Indicator values record progress against the project results framework. Field officers and coordinators submit these regularly (monthly or quarterly, as defined in your project schedule).

### Before You Start

You will need:
- The indicator you are reporting on (ask your coordinator if unsure)
- The value (a number, such as a count or percentage)
- Supporting evidence (a document, photo, or brief description — optional but encouraged)
- The reporting period (e.g., "Q1 2026" or "March 2026")

### Submitting a Value

1. From the sidebar, click **Indicators**.
2. Click **"+ New Submission"** (blue button, top right).
3. Select the **Indicator** from the dropdown list. Indicators are grouped by output and outcome.
4. Select the **Reporting Period** from the calendar or dropdown.
5. Enter the **Actual Value** in the number field.
6. The **Target Value** is shown automatically for reference.
7. (Optional) In the **Notes** field, describe the context: what was measured, any challenges, or anything unusual about this value.
8. (Optional) Click **"Upload Evidence"** to attach a file (photo, PDF, spreadsheet — max 10 MB).
9. Click **"Submit"**.

A green confirmation message will appear. Your submission will be reviewed by your Provincial Coordinator.

### Viewing Your Submissions

1. Click **Indicators** in the sidebar.
2. Your submissions are listed with their status:
   - **Pending** — Submitted, awaiting coordinator review
   - **Approved** — Accepted by the coordinator
   - **Returned** — Sent back for correction (check the Notes for the reason)

### Editing a Returned Submission

1. Click the submission with status **Returned**.
2. Read the coordinator's comment in the **Review Notes** section.
3. Make the required changes.
4. Click **"Resubmit"**.

## 4. Reporting Activities

Activities are actions taken as part of the project: workshops, training sessions, community consultations, field visits, and so on.

### Recording an Activity

1. From the sidebar, click **Activities**.
2. Click **"+ New Activity"**.
3. Fill in the form fields:

   | Field | Description |
   |-------|-------------|
   | Activity Name | Brief name or title |
   | Activity Type | Select from the dropdown (e.g., Training, Consultation, Field Visit) |
   | Date | When the activity took place |
   | Province | Which province this activity was in |
   | Community / Location | The specific village or location name |
   | Implementing Partner | The organisation that carried out the activity |
   | Linked Indicator | (Optional) Which project indicator this activity contributes to |
   | Participants — Male | Number of male participants |
   | Participants — Female | Number of female participants |
   | Participants — Youth | Number of youth participants (under 25) |
   | Description | A brief description of what happened and any outcomes |

4. Click **"Save Activity"**.

### Attaching Outputs

You can attach documents (meeting minutes, photos, sign-in sheets) to any activity:

1. Open the activity record.
2. Click the **"Attachments"** tab.
3. Click **"Upload File"** and select your document.

## 5. Recording Loss and Damage Events

Loss and Damage (L&D) events record climate-related impacts experienced by communities. These are a core data type for the project.

### What to Record

An L&D event is any climate-related incident that caused harm or loss, including:

- Flooding of homes, gardens, or community infrastructure
- Cyclone damage
- Drought affecting crops or water supplies
- Sea level rise or coastal erosion
- Landslides triggered by extreme rainfall

### Recording an Event

1. From the sidebar, click **L&D Events**.
2. Click **"+ New Event"**.
3. Complete the form:

   | Field | Description |
   |-------|-------------|
   | Event Name / Title | Brief description (e.g., "TC Hina flooding — Epi Island") |
   | Hazard Type | Select the type of climate hazard |
   | Start Date | When the event began |
   | End Date | When impacts subsided (leave blank if ongoing) |
   | Province | Affected province |
   | Community | Affected community name |
   | Location | Click on the map to pin the location, or enter GPS coordinates |
   | Severity | Low / Medium / High / Catastrophic |
   | Affected Households | Number of households affected |
   | People Displaced | Number of people who had to leave their homes |
   | Economic Loss (VUV) | Estimated economic value of losses (use 0 if unknown) |
   | Lives Lost | Number of deaths attributed to the event |
   | Description | Detailed description of impacts and community response |
   | Data Source | How this information was collected |

4. Click **"Save Event"**.

### Using the Map to Place an Event

When recording an event, the Location field includes a small map:

1. Click on the map at the approximate location of the event.
2. A marker will appear. Drag it to the precise location if needed.
3. The GPS coordinates will fill in automatically.

## 7. Offline Mode for Community Reporters

Community reporters often work in remote areas without reliable internet. The MERL Dashboard includes an offline mode so you can enter data without a connection and sync it later.

### Setting Up Offline Mode

The first time you use the portal with an internet connection, your browser will install the offline service worker automatically. You do not need to do anything special — just visit the portal once while online.

### Using the Portal Offline

1. Open the MERL Dashboard in your browser (Chrome or Firefox recommended).
2. Navigate to the **Community** or **L&D Events** section before going offline.
3. When your internet connection drops, the portal will show a **"You are offline"** banner.
4. You can continue to fill in forms and click Submit.
5. Instead of sending immediately, the submission is saved to your device with a **"Waiting to sync"** status.

### Syncing When Back Online

1. When you regain internet access, the portal will automatically attempt to sync pending submissions.
2. A notification will appear: **"Syncing X pending records…"**
3. Once sync is complete, the records show status **"Submitted"**.

### Checking Pending Submissions

1. Click your profile icon → **"Pending Sync"** to see a list of records waiting to be uploaded.
2. You can review or delete pending records before they sync.

> **Important:** Do not clear your browser's cache or storage while you have unsynced records. This will permanently delete them.

## 8. Map View Guide

The Map View shows all recorded L&D events, activities, and community engagements plotted on a geographic map of Vanuatu.

### Opening the Map

From the sidebar, click **Map**.

### Navigating the Map

- **Zoom in/out:** Use the +/- buttons, scroll wheel, or pinch gesture on touchscreens.
- **Pan:** Click and drag to move around the map.
- **Reset view:** Click the home icon to zoom back to the full Vanuatu view.

### Layers

Use the Layers panel (top right) to show or hide different data types:

| Layer | Description |
|-------|-------------|
| L&D Events | Red markers for recorded loss and damage events |
| Activities | Blue markers for project activities |
| Community Engagements | Green markers for community meetings |
| Province Boundaries | Provincial boundary polygons |
| Communities | Dots for registered communities |

### Clicking a Marker

Click any marker on the map to see a popup with:
- Event/activity name
- Date
- Key data (affected households, participants, etc.)
- A link to the full record

### Filtering the Map

Use the Filter panel (left side) to narrow down what is shown:
- Filter by date range
- Filter by province
- Filter by event type or hazard type

## 9. Uploading Data via CSV

If you have data in a spreadsheet (Excel or Google Sheets), you can import it in bulk using the CSV upload feature.

### Preparing Your CSV File

1. Export your spreadsheet as a CSV file (.csv format).
2. Ensure the file has a header row (column names in the first row).
3. Remove any merged cells, charts, or extra formatting — CSV files only contain plain text data.

Download a template CSV for each data type:
- **Indicators Template:** From Indicators → Download Template
- **Activities Template:** From Activities → Download Template
- **L&D Events Template:** From L&D Events → Download Template

### Uploading the CSV

1. From the sidebar, click **Upload**.
2. Select the **Data Type** (Indicators, Activities, L&D Events, or Community Engagements).
3. Click **"Choose File"** and select your CSV file.
4. Click **"Upload and Preview"**.

### Mapping Columns

If your column names do not exactly match the template:

1. A column mapping screen will appear.
2. For each system field, use the dropdown to select the matching column from your file.
3. Fields shown in red are required. Fields in grey are optional.
4. Click **"Continue"**.

### Reviewing the Preview

Before confirming the import:

1. A preview table shows the first 10 rows of your data as they will be imported.
2. Check for any red highlighted cells — these indicate validation errors (e.g., invalid date format, missing required value).
3. If there are errors, click **"Cancel"**, fix the CSV, and re-upload.
4. If the preview looks correct, click **"Import X Records"**.

### Import Results

After import:

- A summary shows how many records were successfully imported and how many had errors.
- Successfully imported records appear in the relevant section (Indicators, Activities, etc.) with status **Pending** (awaiting coordinator approval).
- Failed rows are listed with the specific error message — you can download a copy to correct and re-import.

## 10. Downloading Reports

The Reports section allows you to download summaries of MERL data in CSV or PDF format.

### Available Reports

| Report | Description | Format |
|--------|-------------|--------|
| Indicator Progress Summary | All indicators with actual vs target values | CSV, PDF |
| Activity Log | All activities for a selected period | CSV |
| L&D Event Register | All recorded loss and damage events | CSV, PDF |
| Community Engagement Summary | Aggregated engagement statistics | PDF |
| Disaggregated Participation | Gender and youth disaggregation tables | CSV |
| Provincial Comparison | Side-by-side comparison across provinces | PDF |

### Downloading a Report

1. From the sidebar, click **Reports**.
2. Select the **Report Type** from the list.
3. Set the **Date Range** using the date pickers.
4. (Optional) Select a specific **Province** to filter the report.
5. Click **"Generate Report"**.
6. Wait for the report to be prepared (this may take up to 30 seconds for large datasets).
7. Click **"Download CSV"** or **"Download PDF"** when the button appears.

### Superset Dashboards

More advanced analytics and interactive dashboards are available through Apache Superset:

1. From the sidebar, click **Dashboards** (or go directly to `https://merl.climate.gov.vu/superset`).
2. Log in with your MERL credentials.
3. Browse the available dashboards in the Dashboard Gallery.

Superset dashboards update automatically as new data is submitted and approved.
